# Using this skill with DOCX and office-automation skills

This skill should remain the ND30 rulebook. Use it to decide the correct document family, required fields, wording, and compliance rules.

## When to use a generic DOCX skill too

Use a DOCX skill for:
- heavy edits to an existing `.docx`
- redlines, tracked changes, or comments
- OOXML-level fixes
- visual render and layout debugging

Keep this ND30 skill open while doing those tasks so mandatory legal-administrative rules are not lost.

## When to use a general office-automation skill too

Use an office-automation skill for:
- generic Excel transformations
- generic Word template operations not specific to ND30
- large batch data reshaping before ND30 drafting

Keep this skill responsible for:
- choosing the correct ND30 form
- setting required blocks and mandatory labels
- checking signer and recipient logic
- checking attached workbook structure against ND30-related patterns

## Safe combination pattern

1. Decide the ND30 form here.
2. Build or validate the spec here.
3. Use generic DOCX/Excel automation only for convenience work.
4. Return here for final compliance review.
