# ND30 language review and template admission

## LLM review branch

Use the model itself for a second-pass content review after format review.

Check at least:
- spelling and punctuation
- administrative wording and tone
- logic flow between title, trích yếu, căn cứ, phần thân, nơi nhận, and sign-off
- consistency of names, agencies, dates, legal references, abbreviations, and symbols
- suggestions to standardize repetitive wording into a cleaner administrative style

Keep this separate from the ND30 layout report unless the user asks for one combined report.

## Strict template admission workflow

When a user provides a new mẫu and wants the skill to learn from it:

1. Audit the file strictly against ND30 and the intended document family.
2. Summarize fail and warn issues clearly.
3. Ask whether the user still wants to use that mẫu if it still has issues.
4. Only if the user explicitly agrees, register it into the internal template library with its warnings preserved.
5. Later, the skill may suggest that template again, but should mention the stored warnings when they matter.

Do not silently treat a weak mẫu as canonical ND30 truth.
